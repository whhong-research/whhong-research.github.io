Fundamentals of Crack Image Processing: Semantic Segmentation and Post-Processing Techniques

Feb 24, 2025
简体中文

Series Introduction: This series of articles systematically introduces the application of computer vision in structural health monitoring, covering core technologies such as image preprocessing, feature extraction, and deep learning model construction. As the inaugural article of the series, this paper focuses on skeletonization techniques in crack detection and their application in fine-grained segmentation.

1. Task Background and Motivation

1.1 Research Background

In the field of structural health monitoring, crack detection is a critical step in assessing the integrity of concrete structures. Traditional manual inspection methods suffer from limitations such as low efficiency and difficulty in ensuring consistency. In recent years, automated crack detection methods based on computer vision have attracted widespread attention from academia and industry due to their high efficiency and objectivity.

1.2 Technical Approach

The technical approach adopted in this project is as follows:

  1. Image Generation: Utilizing Diffusion models to generate crack-free reference images
  2. Difference Analysis: Computing difference maps between reconstructed and original images using SSIM (Structural Similarity Index)
  3. Coarse Segmentation: Using the difference map as initial crack segmentation results
  4. Fine-grained Segmentation: Inputting segmentation results as prompts into SAM (Segment Anything Model) for refined segmentation

2.2 Advantages of Skeletonization

The SAM model accepts two types of prompts: Mask prompts and Point prompts. Mask prompts can be regarded as dense collections of Point prompts.

Core Idea: Extract centerline feature points of cracks through skeletonization, using them as Prompts for the SAM model to perform fine-grained segmentation.

This approach offers the following advantages:

  • Effectively suppressing noise interference
  • Accurately extracting crack centerlines
  • Generating semantically meaningful prompt points

3. Implementation Code Details

3.1 Code Functionality Overview

  1. Image Preprocessing: Resizing images to (1024, 512) dimensions to facilitate unified definition of morphological denoising kernel sizes and connected component filtering thresholds
  2. Threshold Segmentation: Selecting the darkest 5% of pixels for threshold segmentation, marking these pixels as crack regions
  3. Morphological Operations: Performing morphological opening operations to remove small noise points and deleting connected components with small areas

def adaptive_threshold1(self, image):
    """
    Adaptive threshold segmentation: selecting the darkest 5% of pixels as crack regions
    
    Parameters:
        image: Input image, can be grayscale or color image
    
    Returns:
        binary_img: Binary image with crack regions as white (255) and background as black (0)
    """
    # Resize image to standard dimensions for unified processing
    image = cv2.resize(image, (1024, 512))

    # Convert to grayscale (if input is color image)
    if len(image.shape) > 2:
        gray = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY)
    else:
        gray = image.copy()

    # Bilateral filtering: removing noise while preserving edges
    # d=15: filter neighborhood diameter
    # sigmaColor=150: color space filter sigma
    # sigmaSpace=150: coordinate space filter sigma
    smoothed = cv2.bilateralFilter(gray, d=15, sigmaColor=150, sigmaSpace=150)

    # Flatten image to one-dimensional array
    flat_img = smoothed.flatten()
    
    # Calculate threshold: using the grayscale value of the darkest 5% of pixels as threshold
    # np.percentile calculates the 5th percentile
    threshold = np.percentile(flat_img, 5)
    
    # Binarization: create all-black background
    binary_img = np.zeros_like(smoothed)
    # Set pixels darker than threshold to white (crack regions)
    binary_img[smoothed <= threshold] = 255
    
    return binary_img

def morphological_operations(self, binary):
    """
    Morphological operations: removing noise points and small-area connected components
    
    Parameters:
        binary: Binary image
    
    Returns:
        closed: Image after morphological operations
    """
    # Opening operation: erosion followed by dilation, removing small noise points
    # kernel=(1,1): using 1x1 kernel, minimal morphological operation
    kernel = np.ones((1,1), np.uint8)
    opening = cv2.morphologyEx(binary, cv2.MORPH_OPEN, kernel)
    
    # Connected component analysis: labeling and counting each connected component
    # cv2.connectedComponentsWithStats returns:
    #   num_labels: number of connected components (including background)
    #   labels: connected component label for each pixel
    #   stats: statistical information for each connected component (area, bounding box, etc.)
    #   centroids: centroids of each connected component
    num_labels, labels, stats, centroids = cv2.connectedComponentsWithStats(opening)
    
    # Create new image, keeping only large connected components
    morphed = np.zeros_like(binary)
    
    # Iterate through all connected components (starting from 1, 0 is background)
    for i in range(1, num_labels):
        # If connected component area is greater than or equal to threshold, keep it
        if stats[i, cv2.CC_STAT_AREA] >= self.min_area:
            # Set all pixels of this connected component to white
            morphed[labels == i] = 255
    
    # Closing operation: dilation followed by erosion, filling small holes
    closed = cv2.morphologyEx(morphed, cv2.MORPH_CLOSE, kernel)
            
    return closed

def skeletonize(self, binary):
    """
    Skeletonization: extracting the centerline of cracks
    
    Parameters:
        binary: Binary image
    
    Returns:
        skeleton: Skeleton image with crack centerline as white (255)
    """
    # Ensure input is single-channel binary image
    if len(binary.shape) > 2:
        binary = cv2.cvtColor(binary, cv2.COLOR_BGR2GRAY)      

    # Use skimage skeletonization algorithm
    skeleton = skeletonize(binary)

    # Add post-processing steps (such as removing small branches, etc.)
    skeleton = process_skeleton(skeleton)

    # Convert to uint8 format, white as 255
    skeleton = skeleton.astype(np.uint8) * 255
    
    return skeleton

6.1 cv2.distanceTransform 

1

dst = cv2.distanceTransform(src, distanceType, maskSize[, dstType])

Parameter Description:

  1. src: Input image, must be single-channel 8-bit binary image
  2. distanceType: Distance type
    • cv2.DIST_L1: Manhattan distance
    • cv2.DIST_L2: Euclidean distance
    • cv2.DIST_C: Chebyshev distance
  3. maskSize: Distance transform mask size
    • cv2.DIST_MASK_3: 3x3 mask
    • cv2.DIST_MASK_5: 5x5 mask
    • cv2.DIST_MASK_PRECISE: More precise algorithm
  4. dstType: Output image type (optional)
    • cv2.CV_8U: 8-bit unsigned integer
    • cv2.CV_32F: 32-bit floating point
    • cv2.CV_64F: 64-bit floating point

def medial_axis_skeleton(binary):
    # Return skeleton and distance transform

    return skeleton.astype(np.uint8) * 255, distance
