Aeroelastic Modeling of Wind Turbine Blades: Parameterization Methods for AeroDyn Module

Feb 26, 2025
简体中文

Series Introduction: This series of articles will systematically introduce the usage of OpenFAST wind turbine simulation software, covering the AeroDyn aerodynamics module, ElastoDyn structural dynamics module, control system integration, and more. This article, as the series opener, provides a detailed introduction to the configuration and usage of the AeroDyn aerodynamics module.

Prerequisite Knowledge: The Actuator Line Theory is an important method for wind turbine aerodynamic modeling. It simplifies the blade into line sources distributed along the span, simulating the interaction between the blade and flow field by calculating aerodynamic forces on blade elements. This method maintains computational efficiency while effectively capturing the aerodynamic characteristics of blades.

References

  1. Jonkman, J.M., Hayman, G.J., Jonkman, B.J., Damiani, R.R., Murray, R.E., n.d. AeroDyn v15 User’s Guide and Theory Manual. Renewable Energy.
  2. OpenFAST Official Documentation: https://openfast.readthedocs.io/en/main/index.html

AeroDyn Module Overview

Module Introduction

  1. AeroDyn is a time-domain aerodynamic calculation module that can operate as a standalone program to calculate aerodynamic responses of wind turbines not coupled with OpenFAST, or be integrated into the OpenFAST framework for multi-physics coupled simulations;

  2. This module is primarily used to calculate aerodynamic loads on blades and towers, with its core theoretical foundation being the Actuator Line Theory;

  3. AeroDyn consists of six sub-models, responsible for:

    • Rotor wake and induction velocity calculation (rotor wake/induction)
    • Blade airfoil aerodynamics (blade airfoil aerodynamics)
    • Tower influence on fluid local to blade nodes (tower influence on the fluid local to the blade nodes)
    • Tower and nacelle aerodynamic drag (tower and nacelle drag)
    • Aeroacoustics calculation (aeroacoustics)
    • Buoyancy effects on blades, hub, nacelle, and tower (buoyancy)

Input File Configuration

  1. Basic Unit System: Kilogram-Meter-Second-Newton (kg, m, s, N)

General Options Settings

  1. DTAero: Aerodynamic calculation time step, recommended to set at least 200 steps to ensure computational stability;

  2. Wake_mod (Wake Model Selection):

    • Set to 0: Do not consider induction velocity effects;
    • Set to 1: Calculate induction velocity based on Blade Element Momentum (BEM) theory;
    • Set to 3: Use free vortex wake model, note that this cannot be set to 3 during linear analysis phase;

  3. Twrpotent (Tower Potential Flow Effect):

    • Set to 0: Do not consider tower’s potential influence on local flow field at blades;
    • Set to 1: Use standard potential-flow model;
    • Set to 2: Add Bak correction on top of standard potential-flow model;

Modeling Parameters Recommended Settings

  1. NumVBlNds (Number of Blade Nodes):

    • Recommended to set between 10-20 to balance computational accuracy and cost;

  2. Simulation configuration recommendations for operational wind turbine blades:

    • Recommend enabling dynamic BEM model (Wake_Mod = 1) and unsteady aerodynamics model (AFAeroMod = 2);
    • Enable SkewMod = 2, TipLoss = TRUE, HubLoss = TRUE, TanInd = TRUE;
    • Note: When yaw error exceeds 45 degrees, SkewMod = 2 setting becomes invalid;
    • BEM nonlinear solving is related to inflow angle, IndToler recommended to keep default value DEFAULT;

Output Settings Configuration

  1. AeroDyn supports aerodynamic and kinematic quantity output at 9 nodes along the tower and 9 nodes along the blades;

  2. BldNd_BladesOut: Specifies the number of blades to output results for;

  3. BldNd_BlOutNd: Specifies the node numbers to output;

  4. Blade Node Aerodynamic Force Output Options:

    • “to plane” (relative to rotor plane):

      • Fx: Force perpendicular to blade rotation plane, suitable for evaluating overall blade performance and calculating global loads;
      • Fy: Force parallel to blade rotation plane;

    • “to chord” (relative to blade chord):

      • Fn: Force perpendicular to local blade chord;
      • Ft: Force parallel to local blade chord, in the direction from leading edge to trailing edge, affecting blade torsional deformation;
      • Suitable for analyzing local blade structural response and deformation;

import os
import subprocess
import pandas as pd
import time
from openfast_toolbox.io import FASTInputFile, FASTOutputFile


def split_nodes_into_groups(nodes, group_size=9):
    """将节点列表分成每组最多9个节点"""
    return [nodes[i:i + group_size] for i in range(0, len(nodes), group_size)]


def modify_aerodyn(aero_file_path, node_group, output_path):
    """修改AeroDyn输入文件"""
    f = FASTInputFile(aero_file_path)

    # 设置输出节点数量
    num_nodes = len(node_group)
    f['NBlOuts'] = num_nodes  # 设置输出节点数量

    # 设置节点
    f['BlOutNd'] = ','.join(map(str, node_group))

    # 生成输出列表
    output_list = [""]  # 添加初始换行
    for blade in [1, 2, 3]:
        for node_index, node_value in enumerate(node_group, 1):  # 从1开始计数
            output_list.extend([
                f"B{blade}N{node_index}Fn",
                f"B{blade}N{node_index}Ft",
                f"B{blade}N{node_index}Mm"
            ])

    f['OutList'] = output_list
    f.write(output_path)
    return output_path


def run_openfast(fast_exe_path, fst_file_path):
    """运行OpenFAST"""
    try:
        working_dir = os.path.dirname(fst_file_path)
        os.chdir(working_dir)
        command = [fast_exe_path, os.path.basename(fst_file_path)]
        subprocess.run(command, check=True)
        print("OpenFAST simulation completed successfully")
        return True
    except subprocess.CalledProcessError as e:
        print(f"Error running OpenFAST: {e}")
        return False


def extract_blade_data(outb_file, node_group):
    """从.outb文件提取数据"""
    df = FASTOutputFile(outb_file).toDataFrame()
    blade_data = {1: {}, 2: {}, 3: {}}
    time = df['Time_[s]']

    for blade in [1, 2, 3]:
        data_dict = {'Time_[s]': time}
        # 枚举node_group获取索引和值
        for node_index, node_value in enumerate(node_group, 1):  # 从1开始计数
            for measure in ['Fn', 'Ft', 'Mm']:
                if measure == 'Mm':
                    col_name = f'B{blade}N{node_index}{measure}_[N-m/m]'
                else:  # measure is 'Fn' or 'Ft'
                    col_name = f'B{blade}N{node_index}{measure}_[N/m]'
                if col_name in df.columns:
                    data_dict[f'Node{node_value}_{measure}'] = df[col_name]
                    print(f"Found data for {col_name}")
                else:
                    print(f"Column {col_name} not found")
        blade_data[blade] = pd.DataFrame(data_dict)

    return blade_data


def merge_blade_data(all_runs_data):
    """合并多次运行的数据"""
    merged_data = {1: {}, 2: {}, 3: {}}

    for blade in [1, 2, 3]:
        # 使用第一次运行的时间数据
        merged_df = pd.DataFrame({'Time_[s]': all_runs_data[0][blade]['Time_[s]']})

        # 合并所有运行的数据
        for run_data in all_runs_data:
            cols_to_merge = [col for col in run_data[blade].columns if col != 'Time_[s]']
            merged_df[cols_to_merge] = run_data[blade][cols_to_merge]

        merged_data[blade] = merged_df

    return merged_data


def save_to_excel(blade_data, output_path):
    """保存数据到Excel文件"""
    with pd.ExcelWriter(output_path) as writer:
        for blade_num, data in blade_data.items():
            data.to_excel(writer, sheet_name=f'Blade_{blade_num}', index=False)
    print(f"Data saved to {output_path}")


def main():
    # 设置路径
    work_dir = 'E:/Study/openfast/r-test-3.5.3/glue-codes/openfast/5MW_Test_Weihao'
    fast_exe_path = 'E:/Study/openfast/r-test-3.5.3/glue-codes/openfast/5MW_Test_Weihao/openfast_x64.exe'
    fst_file = os.path.join(work_dir, '5MW_Land_BD_DLL_WTurb.fst')
    aero_file = os.path.join(work_dir, 'NRELOffshrBsline5MW_Onshore_AeroDyn15.dat')
    output_excel = os.path.join(work_dir, 'blade_data_all_nodes.xlsx')

    # 定义所有需要的节点
    all_nodes = list(range(1, 20))
    node_groups = split_nodes_into_groups(all_nodes)

    all_runs_data = []

    for i, node_group in enumerate(node_groups):
        print(f"\nProcessing node group {i + 1}: {node_group}")

        # 修改后的气动文件名
        modified_aero_file = os.path.join(work_dir, f'NRELOffshrBsline5MW_Onshore_AeroDyn15_group_{i + 1}.dat')

        # 修改气动文件
        modify_aerodyn(aero_file, node_group, modified_aero_file)

        # 修改FST文件中的AeroDyn文件路径
        fst = FASTInputFile(fst_file)
        fst['AeroFile'] = f'"{os.path.basename(modified_aero_file)}"'
        modified_fst_file = os.path.join(work_dir, f'5MW_Land_DLL_WTurb_group_{i + 1}.fst')
        fst.write(modified_fst_file)

        # 运行OpenFAST
        if run_openfast(fast_exe_path, modified_fst_file):
            # 提取数据
            outb_file = os.path.join(work_dir, f'5MW_Land_DLL_WTurb_group_{i + 1}.outb')
            run_data = extract_blade_data(outb_file, node_group)
            all_runs_data.append(run_data)

            # 等待一小段时间确保文件写入完成
            time.sleep(1)

    # 合并所有运行的数据
    if all_runs_data:
        merged_data = merge_blade_data(all_runs_data)
        # 保存最终数据
        save_to_excel(merged_data, output_excel)
    else:
        print("No data was collected from any runs")


if __name__ == "__main__":
    main()
  • Pitfall 1: OutList needs an initial newline, otherwise the first variable will not be output;
  • Pitfall 2: BlOutNd needs to use commas to connect variables; direct assignment will result in parentheses, causing errors;
